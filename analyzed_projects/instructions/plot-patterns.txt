# Plot Generation Patterns for ScriptBee Data Model

When generating matplotlib visualizations, use the `generate_plot` MCP tool instead of `execute_code`.
The `generate_plot` tool has both `graph_data` (an `MCPSandboxView` over the typed v2 Graph) and
`plt` (matplotlib.pyplot) pre-imported.

IMPORTANT: Do NOT call plt.show() or plt.savefig() - the server captures the current figure automatically.

NOTE on metric names: examples below use concrete trait/overview/relation names
(`anomaly.testing.BugMagnet`, `components`, `intent_impact`, etc.). These are
illustrations — for the current catalog of names call `list_metrics()`. If an
example references a name that no longer appears there, the metric was renamed;
re-use the pattern with the new name.

---

## Example: Commits per month bar chart

```python
from collections import Counter

commit_counts = Counter()
for commit in graph_data.commits.all():
    if commit.author_date:
        month_label = commit.author_date.strftime('%Y-%m')
        commit_counts[month_label] += 1

months = sorted(commit_counts.keys())
counts = [commit_counts[m] for m in months]

plt.figure(figsize=(10, 5))
plt.bar(months, counts)
plt.title('Commits per Month')
plt.xlabel('Month')
plt.ylabel('Number of Commits')
plt.xticks(rotation=45)
plt.tight_layout()
```

## Example: Top 10 contributors horizontal bar chart

```python
from collections import Counter

author_counts = Counter()
for commit in graph_data.commits.all():
    # `.author(graph_data)` is the generated resolver for `author_ref`.
    author = commit.author(graph_data)
    if author is not None:
        author_counts[author.name] += 1

top_authors = author_counts.most_common(10)
names, counts = zip(*top_authors) if top_authors else ([], [])

plt.figure(figsize=(10, 6))
plt.barh(list(reversed(names)), list(reversed(counts)))
plt.xlabel('Commits')
plt.title('Top 10 Contributors')
plt.tight_layout()
```

## Example: Issue types pie chart

```python
from collections import Counter

type_counts = Counter()
# `.type(graph_data)` is the generated resolver for `type_ref`.
for issue in graph_data.issues.all():
    issue_type = issue.type(graph_data)
    if issue_type is not None:
        type_counts[issue_type.name] += 1

labels = list(type_counts.keys())
sizes = list(type_counts.values())

plt.figure(figsize=(8, 8))
plt.pie(sizes, labels=labels, autopct='%1.1f%%', startangle=90)
plt.title('Issue Types Distribution')
plt.tight_layout()
```

## Example: BugMagnet files bar chart (using enrichment layer)

```python
# `find_files_with_trait` returns File entities; pull severity / evidence
# off the matching Trait via graph_data.traits.for_target(file.ref()).
files = find_files_with_trait("anomaly.testing.BugMagnet")

top = []
for f in files:
    traits = graph_data.traits.for_target(f.ref())
    bm = next((t for t in traits if t.name == "anomaly.testing.BugMagnet"), None)
    if bm is None:
        continue
    top.append((f.id, bm.evidence.get("bugfix_ratio", 0.0)))
top.sort(key=lambda x: -x[1])
top = top[:10]

names, values = zip(*top) if top else ([], [])
plt.figure(figsize=(10, 5))
plt.barh(list(reversed(names)), list(reversed(values)))
plt.xlabel("Bugfix Ratio")
plt.title("Top 10 BugMagnet Files")
plt.tight_layout()
```

## Example: Components churn comparison

```python
table = overview_as_dict("components")
labels = []
churn = []
for entity, cells in table["rows"].items():
    if entity == "(project)":
        continue
    v = cells.get("total_churn", {}).get("lifetime_value")
    if v in (None, "", 0):
        continue
    labels.append(entity)
    churn.append(float(v))

# Top 10 most-churning components.
order = sorted(range(len(labels)), key=lambda i: -churn[i])[:10]
labels = [labels[i] for i in order]
churn = [churn[i] for i in order]

plt.figure(figsize=(10, 5))
plt.barh(list(reversed(labels)), list(reversed(churn)))
plt.xlabel("Lifetime Churn")
plt.title("Top 10 Components by Churn")
plt.tight_layout()
```

## LOC vs. churn scatter (Lizard B1)

```python
# Pair each Lizard sum_nloc row with the matching v2 File, then count
# the file's lifetime added+deleted lines via the change->hunk chain.
# In v2 FileMetric is one (file_ref, metric_name, value) row per metric;
# filter by metric_name == "sum_nloc" to pick the LOC rows.
xs, ys, labels = [], [], []
for m in graph_data.file_metrics:
    if m.metric_name != "sum_nloc":
        continue
    # Resolve the file via the EntityRef the metric carries — id IS the path.
    git_file = graph_data.files.get(m.file_ref.id)
    if git_file is None:
        continue
    churn = 0
    for change in graph_data.changes.by_file[git_file.ref()]:
        for hunk_ref in change.hunk_refs:
            hunk = graph_data.hunks.get(hunk_ref.id)
            if hunk is None:
                continue
            churn += len(hunk.added_lines) + len(hunk.deleted_lines)
    xs.append(m.value)
    ys.append(churn)
    labels.append(m.file_ref.id)

plt.figure(figsize=(8, 6))
plt.scatter(xs, ys, alpha=0.5)
plt.xlabel("LOC (Lizard NLOC)")
plt.ylabel("Lifetime churn (added + deleted)")
plt.title("LOC vs. churn — DynamicBlob lives in the upper right")
plt.tight_layout()
```

## Tips

- Always use `plt.figure(figsize=(w, h))` to set a reasonable size
- Use `plt.tight_layout()` to prevent label clipping
- For date axes, use `plt.xticks(rotation=45)` for readability
- Standard library modules (collections, datetime) are available
- The server returns JPEG - use clear colors with good contrast
- Read-only MCP wrappers (`list_anomalies`, `get_overview_table`,
  `get_relation_edges`) fetch data directly without writing /execute code,
  for quick chart inputs. Call `list_metrics()` first to know the right
  trait_name / overview_name / relation_kind to pass.
- For relation-based plots, prefer `graph_data.relations.of_kind_in_window(kind, window)`
  over the legacy `enrichments.relation_file(...)` shape — the v2 registry
  has the strength on each `Relation` directly.
